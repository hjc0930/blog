# Spring Boot 练习项目推荐方案

## 项目：个人博客系统

### 为什么推荐这个项目
- **功能完整**：涵盖增删改查、用户认证、文件上传等核心功能
- **难度适中**：不会太简单，也不会过于复杂
- **实用性强**：类似真实业务场景

---

## 核心功能模块

### 1. 用户模块
- 用户注册/登录（JWT认证）
- 个人信息修改
- 密码重置

### 2. 文章模块
- 文章发布（Markdown编辑器）
- 文章列表（分页、分类筛选）
- 文章详情
- 文章编辑/删除
- 文章点赞/收藏

### 3. 评论模块
- 发表评论
- 评论回复（二级评论）
- 评论删除

### 4. 分类/标签模块
- 分类管理
- 标签管理
- 按分类/标签查询文章

### 5. 文件上传模块
- 头像上传
- 文章封面图上传

---

## 技术栈建议

| 层次 | 技术 |
|------|------|
| 后端框架 | Spring Boot 3.x |
| 数据库 | MySQL + MyBatis-Plus |
| 缓存 | Redis |
| 安全认证 | Spring Security + JWT |
| 文档 | Knife4j/Swagger |
| 工具库 | Lombok, Hutool |

---

## 练习重点

### 第一阶段：基础 CRUD
- 搭建项目骨架
- 完成用户、文章的基础增删改查
- 统一响应结果封装
- 全局异常处理

### 第二阶段：高级功能
- 集成 Redis 缓存热门文章
- 实现 JWT 登录认证
- 文件上传（阿里云 OSS 或本地存储）
- 分页查询、条件筛选

### 第三阶段：进阶优化
- 日志记录（AOP）
- 接口限流
- 定时任务（文章定时发布）
- 单元测试

---

## 数据库表设计建议

1. **user** - 用户表
2. **article** - 文章表
3. **category** - 分类表
4. **tag** - 标签表
5. **article_tag** - 文章标签关联表
6. **comment** - 评论表
7. **like** - 点赞表

---

## 项目结构建议

```
com.example.blog
├── controller      # 控制层
├── service         # 业务层
├── mapper          # 数据访问层
├── entity          # 实体类
├── dto             # 数据传输对象
├── vo              # 视图对象
├── config          # 配置类
├── common          # 公共模块
│   ├── result      # 统一响应
│   └── exception   # 异常处理
└── utils           # 工具类
```

---

这个项目足够练习 Spring Boot 的核心技能，完成后可以根据兴趣继续扩展（如添加搜索功能、邮件通知等）。
